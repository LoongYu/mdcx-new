## 新增

* 新增 `jable` 主站点支持，支持有码刮削、站点配置、网络检测与自定义域名
* 新增 `jable` 单文件刮削支持，可直接输入 `https://jable.tv/videos/...` 详情页地址

## 变更

* `jable` 搜索页与详情页请求统一追加 `?lang=zh`，优先使用中文页面结构

## 修复

* 修复单文件手工指定 `jable` 详情页时，仍因本地提取番号与页面番号扩展后缀不同而误判失败的问题

## 修复

* `madou` 单文件刮削在手工指定详情页 URL 时，改为直接信任人工确认的详情页，不再因本地提取番号与页面番号扩展后缀不一致而误判失败

## 新增

* 主界面新增“读取已刮削”按钮，支持手动选择任意已刮削目录，递归加载本地 `NFO` 结果并回显到主界面
* 已刮削目录读取结果按“一个番号显示一条”聚合，成功列表仅显示番号，便于后续集中检查与编辑

## 变更

* 主界面编辑并保存 `NFO` 时，文件夹与文件重命名逻辑扩展为覆盖全部命名字段，不再仅限演员字段
* 设置页关键下拉弹层增加鼠标悬停高亮，并限制弹层显示高度，超出部分通过滚动查看

## 修复

* 修复主界面编辑并保存 `NFO` 后，封面预览未按新路径重新刷新的问题
* 修复命名模板字段为空时回填 `0000-00-00`、`0000` 等占位内容的问题，改为空值不参与命名
* 修复设置页“网站偏好 / 当前网站”下拉列表、网络滑块与数字显示控件在浅色主题下的背景色异常

## 新增

* 新增国产站点 `madou`，支持工具单文件刮削、国产站点优先级配置与详情页精确匹配
* 主界面编辑 NFO 后，保存时支持按最新命名字段自动同步重命名归类文件夹、视频文件、NFO、图片、字幕与 trailer

## 修复

* 修复 `madou` 图片资源需要前端解密导致下载后图片损坏的问题，增加自动解密还原
* 修复 `madou` 误取播放器预览首帧作为封面的问题，改为优先使用页面真实封面
* 修复主界面编辑并保存 NFO 后，封面预览未按新路径重新加载的问题

## 新增

* 网络检测支持并发执行（受控并发），显著降低整轮检测耗时
* 网络检测统一支持“自定义域名优先，默认域名兜底”策略
* 配置目录切换与配置恢复支持动态选择配置文件，不再固定 `config.json`
* `javday` 新增从“上傳時間/上传时间”提取发布时间（`release`）与年份（`year`）
* `javday` 新增 `/videos/` 详情直连优先策略，并支持自动尝试“去分隔符”slug（如 `MD0356`、`MDSR00131`）

## 变更

* 清理失效刮削站点：`oppai`、`v-av`、`to-satsu`、`bibian`、`nanpa`、`mko-labo`、`muku`、`befreebe`、`prestige`、`faleno`、`yesjav`、`love6`、`fc2club`、`airav`
* 继续清理失效站点：`avsex`、`avsox`
* 同步移除对应配置项、网络检测列表、站点映射与 UI 选项，避免残留引用
* 站点禁用策略统一收敛到 `DISABLED_WEBSITES`，旧配置会自动过滤禁用站点并回退到可用默认值
* 发布流程支持在手动输入更新日志时，将 `\n` 自动转换为真实换行

## 修复

* 修复旧配置中残留失效站点导致的网站设置异常、网络检测异常与配置校验失败
* 修复 Windows 发布包命名尾部附带 SHA ID 的问题（现为简洁文件名）
* 修复 Windows 发布未正确带入 changelog 的问题
* 修复 `javday` 直接拼接 `/videos/{number}` 导致的误匹配与超时，改为按 `/search/?wd=` 搜索再进入详情页
* 修复 `javday` 详情页封面链接提取兼容性（优先 `og:image`，保留兜底）
* 修复 `javday` 在 `search` 返回 HTTP 403（Cloudflare）时立即失败的问题，增加多路兜底与详情页回退
* 修复 `javday` 受文件名噪声（如 `cctv666.com`）干扰导致候选番号顺序错误的问题
* 修复 `javday` 三段番号与连字符变体在详情直连时命中率低的问题
* 修复 `javlibrary`、`iqqtv` 等触发 Cloudflare 场景被误判为断网的问题（改为可访问但受限提示）
* 修复网络检测在代理链路异常场景下的误判，增加直连兜底判定
* 修复网站设置中自定义域名切换后易丢失、检测仍走旧默认域名的问题
* 修复配置恢复与目录切换逻辑写死 `config.json` 导致多配置文件场景异常的问题
